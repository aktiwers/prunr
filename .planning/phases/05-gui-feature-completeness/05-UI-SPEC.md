## Interaction Contract

### State Machine Extension

Phase 5 adds one new AppState variant and extends batch awareness.

```
Empty → Loaded (single image) → Processing → Done
                              ↘ Animating → Done   ← NEW: after Processing completes
Loaded (multi image) → sidebar visible automatically
```

| State | Canvas | Sidebar | Status bar | Toolbar |
|-------|--------|---------|------------|---------|
| Empty | Drop zone hint (inherited) | Hidden | "Ready" + backend badge | inherited |
| Loaded (single) | Image at fit zoom (inherited) | Hidden | "Ready" + dimensions + zoom% + backend | inherited |
| Loaded (multi) | Selected image at fit zoom | Thumbnail strip, auto-shown | "Ready" + dimensions + zoom% + backend | Process All enabled |
| Processing (single) | Dimmed image + "Press Escape to cancel" | — | Progress bar + stage + Escape hint | inherited |
| Processing (batch) | Selected image dimmed if processing | Sidebar with ◆ icons animating | "Processing {N}/{total}..." | Process All disabled |
| Animating | Reveal animation rendered over canvas | — | "Done" | Save/Copy enabled |
| Done | Result image with checkerboard | Result shown if processed | "Done" + dimensions + zoom% + backend | inherited |

**Before/After toggle:** Active only in Done state. B key toggles `show_original: bool`. Indicator label ("Original" or "Result") shown in top-left of canvas at 12px Monospace, `#888888` text.

### Zoom and Pan

- **Zoom range:** 10% to 2000% (source: CONTEXT.md)
- **Zoom step:** multiply by `1.1` per scroll tick (source: CONTEXT.md "approximately 1.1x")
- **Zoom direction:** scroll up = zoom in, scroll down = zoom out
- **Cursor centering:** zoom origin is pointer hover position at time of scroll event
- **Pan:** Space held + primary button drag; cursor changes to grab cursor while Space held
- **Ctrl+0:** fit-to-window; if already at fit zoom (within 0.001 tolerance), restore `previous_zoom`
- **Ctrl+1:** 1:1 pixel zoom (zoom = 1.0); if already at 1.0 (within 0.001 tolerance), restore `previous_zoom`
- **Reset on new image:** zoom = fit, pan_offset = Vec2::ZERO, previous_zoom = 1.0
- **Zoom display:** live percentage in status bar, right-aligned before backend badge

### Reveal Animation

- **Trigger:** immediately after AppState transitions from Processing to Done (if animation enabled in settings)
- **Duration:** 800ms total (midpoint of the 0.5–1s range from CONTEXT.md)
- **Easing:** ease-out cubic (`t = 1 - (1-t)^3`) — smooth deceleration feels like settling
- **Mechanism:** per-pixel alpha on background region decreases from 255 to 0 driven by `stable_dt` accumulator
- **Subject pixels** (alpha > 128 in result mask): always rendered at 100% opacity — never touched
- **Background pixels** (alpha <= 128 in result mask): alpha interpolates from original source image alpha to 0; checkerboard reveals beneath
- **Skip:** any key press OR any mouse click immediately jumps animation to final Done state (source: CONTEXT.md)
- **No skip hint:** no text overlay during animation (source: CONTEXT.md)
- **Settings gate:** if `Settings::reveal_animation == false`, skip immediately to Done state; no AppState::Animating entered
- **Single play:** animation plays once per processing completion; switching before/after does not replay it

### Batch Sidebar

- **Auto-show threshold:** 2 or more images in queue (source: CONTEXT.md)
- **Manual toggle:** Tab key and "Show/Hide Queue" toolbar affordance
- **Position:** `Panel::left`, width = 96px (SIDEBAR_WIDTH), full height below toolbar
- **Thumbnail:** 80px square letterboxed image; status icon overlaid in bottom-right corner at xs offset (4px inset)
- **Status icons:** ○ = pending (#888888), ◆ = processing (#3b82f6 pulsing opacity 60%–100% at 1Hz), ✓ = done (#22c55e)
- **Selected state:** 2px accent border (#3b82f6) around thumbnail cell, slightly lighter cell background (#2a2a2a)
- **Click:** sets `selected_index`; canvas immediately renders that item's result (if done) or source (if pending)
- **Keyboard navigation:** [ = previous (wraps), ] = next (wraps) (source: REQUIREMENTS.md UX-05)
- **Drag-to-reorder:** egui 0.34 native DnD (`dnd_set_drag_payload` / `dnd_release_payload`); ghost = 50% alpha copy of thumbnail; insertion line = 2px accent line between items
- **Sidebar empty state:** shown when queue is empty — "Drop images here to queue them" at 12px TEXT_SECONDARY, centered vertically in panel
- **Batch progress:** ◆ icon pulses; overall count in status bar

### Process All

- **Trigger:** toolbar button labeled "Process All" (left of spacer, after "Remove BG"); keyboard shortcut none (source: CONTEXT.md — "Claude's discretion on how triggered" — toolbar button chosen)
- **Behavior:** calls `batch_process()` on all items where `result.is_none()`; skips already-processed items
- **Button state:** disabled if all items already processed or if batch is running
- **Parallelism:** from `Settings::parallel_jobs` (source: CONTEXT.md)
- **Cancel:** Escape cancels running batch; same cancel flag pattern as Phase 4

### Settings Dialog

- **Trigger:** Ctrl/Cmd+, (source: CONTEXT.md)
- **Dismiss:** Escape key OR × button in top-right corner
- **Position:** centered modal using `egui::Window::new("Settings").anchor(Align2::CENTER_CENTER, Vec2::ZERO)`
- **Background overlay:** same pattern as shortcuts overlay — `#1a1a1a` at 95% alpha over full window
- **Dialog size:** 400×320px fixed (SETTINGS_DIALOG_WIDTH × SETTINGS_DIALOG_HEIGHT)
- **Border:** 1px `#ffffff20`, 8px corner_radius
- **Layout:** title row + separator + 5 field rows at 16px vertical spacing each

Settings field layout order (top to bottom):

| # | Field | Widget | Default |
|---|-------|--------|---------|
| 1 | Model | ComboBox (egui) with "silueta (fast, ~4 MB)" / "u2net (quality, ~170 MB)" | silueta |
| 2 | Auto-remove on import | Checkbox | unchecked |
| 3 | Parallel jobs | Slider, range 1 to num_cpus::get(), integer steps | num_cpus / 2, min 1 |
| 4 | Reveal animation | Checkbox | checked |
| 5 | Inference backend | Read-only text label (12px Monospace) | detected at runtime |

- Field labels: left-aligned, 14px Body, TEXT_PRIMARY; field widgets: right-aligned in same row
- Field descriptions: one line below field in 12px TEXT_SECONDARY (shown only where listed in Copywriting Contract above)
- No OK/Cancel buttons — changes apply immediately (egui immediate-mode pattern)
- Settings struct must derive `serde::Serialize` / `serde::Deserialize`; file persistence deferred to Phase 6
