# Plan 07-03: UI Polish — History Indicator, Depth Slider, Status Bar

<objective>
Show the user how deep they are in the history stack, add a history depth slider to settings, and update the toolbar/status bar to reflect chain mode state.
</objective>

## Tasks

<task id="1">
<title>Add history depth slider to General settings tab</title>
<read_first>
- crates/prunr-app/src/gui/views/settings.rs (General tab)
</read_first>
<action>
After the chain_mode checkbox in General tab, add (only visible when chain_mode is true):

```rust
if app.settings.chain_mode {
    ui.add_space(theme::SPACE_SM);
    let depth_text = format!("{}", app.settings.history_depth);
    slider_row(
        ui, "History depth", &mut (app.settings.history_depth as f32),
        1.0..=50.0, &depth_text, false, Some(1.0),
    );
    hint(ui, "Maximum undo steps per image. Higher = more memory.");
}
```

Note: slider_row takes f32, cast back to usize on change.
</action>
<acceptance_criteria>
- History depth slider visible when chain_mode is true
- Slider range 1-50
- Value displays as integer
- Hidden when chain_mode is false
</acceptance_criteria>
</task>

<task id="2">
<title>Show history depth indicator in status bar</title>
<read_first>
- crates/prunr-app/src/gui/views/statusbar.rs
- crates/prunr-app/src/gui/app.rs (selected_item)
</read_first>
<action>
In the status bar, when chain_mode is enabled and the current image has history, show a small indicator like "3/10 layers" or a chain icon:

```rust
if app.settings.chain_mode {
    if let Some(item) = app.selected_item() {
        let depth = item.history.len();
        if depth > 0 {
            ui.add_space(theme::SPACE_SM);
            ui.label(
                RichText::new(format!("{depth} undo"))
                    .monospace()
                    .size(theme::FONT_SIZE_MONO)
                    .color(theme::TEXT_SECONDARY),
            );
        }
    }
}
```
</action>
<acceptance_criteria>
- Status bar shows undo count when chain_mode is on and history is non-empty
- Not shown when chain_mode is off
- Not shown when history is empty
</acceptance_criteria>
</task>
